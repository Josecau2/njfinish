✅ Best Practices - Form Modals
  • Validation: Inline error messages below fields, not alerts
  • Required Fields: Mark with * and validate on blur
  • Primary Action: Always on the right, colored (accent)
  • Cancel Action: Always on the left, neutral (gray)
  • Loading States: Disable all inputs, show spinner in button
  • Tab Order: Input → Input → Checkbox → Cancel → Submit
  • Mobile: Buttons stack vertically (primary on top)

✅ Best Practices - Confirmation Modals
  • Warning Alert: Always show with yellow/red indicator
  • Explicit Message: State exactly what will happen
  • Destructive Action: Red button, on the right

Global Styling Rules

Consistent patterns across all modals

🎨 Color System
  • Header Background: #000000 (brand.headerBg)
  • Header Text: #ffffff with #808080 for close button
  • Primary Action: #2563eb (brand.accent)
  • Destructive Action: #dc2626 (red-600)
  • Text: #0f172a (slate-900)
  • Border: #e2e8f0 (slate-200)
  • Overlay: rgba(0, 0, 0, 0.48)
📐 Spacing System
  • Base Unit: 8px (0.5rem)
  • Header Padding: 20px × 24px (desktop), 16px × 20px (mobile)
  • Body Padding: 24px (desktop), 20px (mobile)
  • Footer Padding: 16px × 24px (desktop), 20px (mobile)
  • Button Gap: 12px between buttons
  • Form Group Margin: 20px between form groups
⌨️ Keyboard Accessibility
  • Escape Key: Closes modal (except confirmations)
  • Tab Key: Cycles through focusable elements
  • Enter Key: Submits form (on primary button)
  • Focus Trap: Prevents tabbing outside modal
  • Auto-Focus: First input gets focus on open
🔧 Component Patterns
  • Loading State: Disable all inputs, show spinner in button, gray out
  • Error State: Red border on input, error message below, shake animation
  • Success State: Green checkmark, brief confirmation, auto-close after 2s
  • Disabled State: Gray background, 60% opacity, cursor: not-allowed
  • Required Fields: Red asterisk (*) after label
  • Helper Text: Gray text, 12px font, below input

Implementation Checklist

Apply these patterns to Chakra UI modals

✅ For Every Modal
  • ✅ Use scrollBehavior="inside" to fix header/footer
  • ✅ Set size={{ base: 'full', md: 'md/lg/xl/6xl' }} for responsive sizing
  • ✅ Use isCentered for desktop centering
  • ✅ Apply customization.headerBg to ModalHeader
  • ✅ Calculate contrast color for header text via getContrastColor()
  • ✅ Mobile: Footer buttons stack vertically with flex-direction: column-reverse
  • ✅ Desktop: Footer buttons inline with justify-content: flex-end
  • ✅ Primary button uses customization.primaryColor or accent
  • ✅ Add aria-label="Close modal" to close button
  • ✅ Disable close on overlay click for confirmations: closeOnOverlayClick={false}
🚫 Common Mistakes to Avoid
  • ❌ Hardcoded colors (use theme variables)
  • ❌ Fixed pixel sizes (use responsive breakpoints)
  • ❌ Scroll entire modal (use scrollBehavior="inside")
  • ❌ Same button order on mobile/desktop (reverse on mobile)
  • ❌ Missing loading states (always show spinner in buttons)
  • ❌ No keyboard support (add focus trap and Escape key)
  • ❌ Inconsistent padding (use spacing system)
